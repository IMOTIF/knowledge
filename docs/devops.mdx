## Intro & Overview

ในโลกการพัฒนาซอฟต์แวร์จริง การเขียนโค้ดให้รันได้ในเครื่องตัวเองยังไม่ถือว่างานเสร็จ
โค้ดหนึ่งชุดต้องผ่านหลายขั้นตอน ตั้งแต่การพัฒนา การทดสอบ ไปจนถึงการนำไปใช้งานจริง (Production)

การอบรมนี้ออกแบบมาเพื่อให้น้อง ๆ เข้าใจ **ภาพรวมของการทำงานแบบทีม**
และเห็นเส้นทางของโค้ดตั้งแต่วันแรกที่เขียน จนถึงวันที่ผู้ใช้งานสามารถเข้าถึงได้จริง

---

## Dev / Test / Production Workflow

กระบวนการทำงานทั่วไปสามารถแบ่งออกเป็น 3 ส่วนหลัก:

### Dev (Development)

* ใช้สำหรับเขียนและทดลองโค้ด
* สามารถแก้ไขและทดลองได้อย่างอิสระ
* ความผิดพลาดไม่กระทบผู้ใช้งานจริง

### Test / Staging

* ใช้สำหรับตรวจสอบโค้ดก่อนปล่อยจริง
* มีการรัน automated tests
* สภาพแวดล้อมใกล้เคียง production มากขึ้น

### Production

* ระบบที่เปิดให้ผู้ใช้งานจริงเข้าถึง
* ต้องมีความเสถียรและปลอดภัยสูง
* ความผิดพลาดส่งผลกระทบโดยตรงต่อผู้ใช้งาน

---

## Continuous Integration (CI)

CI คือกระบวนการตรวจสอบโค้ดอัตโนมัติทุกครั้งที่มีการเปลี่ยนแปลง
เช่น การรัน test หรือการตรวจรูปแบบโค้ด เพื่อให้มั่นใจว่าโค้ดใหม่ไม่ทำให้ระบบเดิมพัง

ในคอร์สนี้จะใช้ **GitHub Actions** เป็นเครื่องมือ CI
โดยเมื่อมีการ push โค้ด ระบบจะรัน **Jest tests** ให้อัตโนมัติ

---

## Continuous Deployment (CD)

CD คือขั้นตอนนำโค้ดที่ผ่านการตรวจสอบแล้วไป deploy
อาจเป็นการ deploy แบบกดเอง หรือแบบอัตโนมัติ

ในการอบรมนี้จะใช้ **Docker** และ **Watchtower**
เพื่อแสดงให้เห็นแนวคิดของการ deploy ที่ลดขั้นตอน manual และลดความผิดพลาดจากมนุษย์

---

## Git

Git คือระบบควบคุมเวอร์ชัน (Version Control System)
ที่ใช้จัดการการเปลี่ยนแปลงของโค้ด และช่วยให้หลายคนทำงานร่วมกันได้อย่างเป็นระบบ

Git ทำงานอยู่ในเครื่องของผู้พัฒนา และสามารถใช้งานได้แม้ไม่เชื่อมต่ออินเทอร์เน็ต

---

## การเริ่มต้นใช้งาน Git

### `git init`

ใช้สำหรับเริ่มต้น repository ใหม่ในโฟลเดอร์ปัจจุบัน

```bash
git init
```

เหมาะสำหรับโปรเจกต์ที่เริ่มจากศูนย์

---

### `git clone`

ใช้สำหรับดึง repository ที่มีอยู่แล้วมาจาก remote

```bash
git clone <repository-url>
```

เหมาะสำหรับโปรเจกต์ที่ทำงานเป็นทีม หรือโปรเจกต์จาก GitHub

---

## การบันทึกการเปลี่ยนแปลง

### `git add`

เลือกไฟล์ที่ต้องการบันทึกการเปลี่ยนแปลง

```bash
git add .
```

---

### `git commit`

บันทึกสถานะของโค้ด ณ เวลานั้น พร้อมข้อความอธิบาย

```bash
git commit -m "Add login feature"
```

ข้อความ commit ควรอธิบายว่า “ทำอะไร” ไม่ใช่ “แก้อะไรไปบ้าง”

---

### `git log`

แสดงประวัติการ commit ทั้งหมด

```bash
git log
```

---

## Branch เบื้องต้น

Branch คือเส้นทางการพัฒนาแยกออกจากโค้ดหลัก
ช่วยให้สามารถทดลองหรือพัฒนา feature ใหม่ได้โดยไม่กระทบโค้ดหลัก

### สร้าง branch ใหม่

```bash
git branch feature/login
```

### สลับไปยัง branch

```bash
git checkout feature/login
```

หรือ

```bash
git switch feature/login
```
